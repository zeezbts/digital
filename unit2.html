1. Commutative Law
    Definition: Changing the order of the inputs in a logical operation produces the exact same output.
    OR Law:
    A + B = B + A
    AND Law:
    A · B = B · A
    Commutative Law

    Commutative Law Diagrams

    A + B = B + A
    ≡
    A · B = B · A
    ≡
  2. Associative Law
    Definition: The grouping of variables does not change the final output when performing repeated operations of the same type.
    OR Law:
    A + (B + C) = (A + B) + C
    AND Law:
    A · (B · C) = (A · B) · C
    Associative Law

    Associative Law Diagrams

    A + (B + C) = (A + B) + C
    ≡
    A · (B · C) = (A · B) · C
    ≡
  3. Distributive Law
    Definition: An operation can be applied across a parenthetical group of variables, similar to multiplying across an addition in standard algebra.
    AND over OR:
    A(B + C) = AB + AC
    OR over AND:
    A + BC = (A + B)(A + C)
    Distributive Law

    Distributive Law Diagrams

    A(B + C) = AB + AC
    ≡
    A + BC = (A + B)(A + C)
    ≡
  4. De Morgan's Laws
    Definition: The complement of a grouped logical expression is found by breaking the inversion bar over each individual variable and flipping the operator sign underneath.
    First Law:
    A + B = A · B (The complement of a sum equals the product of the complements)
    Second Law:
    A · B = A + B (The complement of a product equals the sum of the complements)
    De Morgan's Laws

    De Morgan's Laws Diagrams

    A + B = A · B
    ≡
    A · B = A + B
    ≡

    3. It is sum of minterms. Minterms are represented as 'm' It is product of maxterms. Maxterms are represented as 'M'
    4. SOP is formed by considering all the minterms, whose output is HIGH(1) POS is formed by considering all the maxterms, whose output is LOW(0)

    Binary Representations Specification

    1. Binary Representations

    Binary representations dictate how positive, negative, and fractional values are stored in binary memory fields.

    • Unsigned Representation

      All bits represent positional magnitude weights (e.g., 2n, ..., 22, 21, 20). It cannot represent numbers less than zero.

    • Sign-Magnitude (SM)

      The Most Significant Bit (MSB) acts as the sign flag. The remaining bits determine absolute value.

      • Sign Bit: 0 = Positive (+), 1 = Negative (-)
      Example (+5 vs -5 in 4 bits):
      +5 → 0101 | -5 → 1101
    • 1's Complement

      Formed by flipping every bit of the positive number (1 becomes 0, and 0 becomes 1).

      Example (-5 in 4 bits):
      +5 = 0101 → -5 = 1010
    • 2's Complement

      The standard system for modern processors. Formed by adding 1 to the 1's complement representation. It provides a single representation for zero and simplifies hardware subtraction.

      Example (-5 in 4 bits):
      +5 = 0101 → 1's Comp = 1010 → Add 1 → -5 = 1011
